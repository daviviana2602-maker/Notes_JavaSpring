=============== Annotations automáticas ===============


@Service, @Controller, @Repository, @Entity @Configuration

“eu achei essa classe, vou criar ela sozinho”

Spring faz tudo automaticamente via scan


Na prática:

    @Service
    é um @Component para regras de negócio.

    @Repository
    é um @Component para acesso ao banco.

    @Controller`
    é um @Component para controllers MVC.

Ou seja, todos eles acabam sendo componentes gerenciados pelo Spring.



@Component

Diz pro Spring:

“cria e gerencia essa classe automaticamente”



@Bean

@Bean dentro de uma classe @Configuration

“eu estou criando manualmente um objeto e entregando ele para o Spring somente gerenciar”

Você cria o objeto manualmente e entrega pro Spring



@Transactional

Abre uma transação no banco de dados.

Use em métodos que realizam alterações no banco (save, update ou delete).

Se todas as operações forem concluídas com sucesso, a transação é confirmada (commit).

Caso ocorra uma exceção durante a execução, todas as alterações realizadas na transação são desfeitas (rollback), evitando que o banco fique em um estado inconsistente.
